用hexo和Typora写博客时遇到的时区问题

​

非常离谱的一个error，耗费了我很多时间才初步解决

问题

在下载hexo与GitHub连接并配置设置时，_config.yml文件里，我设置的时区timezone是Asia/Beijing,

之后在尝试发表我的第一篇博客时就频频报错，在hexo g 生成博客时就报错，error发给文心一言说是时区设置错误，网上搜索了一下我自己的时区应该是Asia/Shanghai,修改后又该了一下其他的东西总算是发出成功了

但是隔天第二次发布博客又遇到同样的问题。在ERROR后还有几十行如下代码

Moment Timezone has no data for Asia/Beijing. See http://momentjs.com/timezone/docs/#/data-loading/.

上面这行代码在git bash中重复了几十行，不明白原因，明明我已经把时区改成Shanghai了，也保存了。

解决

本来以为是设置的问题，但我把第一篇博客内容复制粘贴到第二篇上，第二篇就又可以发出去了，判断是内容的问题，但内容的问题最开始就考虑过，文章的格式什么的也都注意，甚至连中英文空格都试了（虽然很扯）。最后发现是什么问题，都给我整懵了。

是当你使用hexo new name 生成md文件的时候，打开name.md上面默认生成的

title: generate_hexo
date: 2024-05-29 08：23：48
tags: hexo

这个有问题，而且这个的问题真的很难发现。

需要把date具体时间中的冒号从英文改成中文！

它默认是生成英文冒号: ，而英文冒号又和中文冒号：很像很像。但两者不一样！

真的想不出来出现这样的问题怎么发现。
